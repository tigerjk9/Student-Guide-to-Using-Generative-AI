# PDF → 콘텐츠 맵

인쇄물의 "내용과 학습의도"를 웹 변환에 적합한 형태로 추출한다. 디자인·구현은 다루지 않는다.

## 추출 절차
1. **페이지 수·메타 확인** → 목차 페이지에서 섹션 위계 파악.
2. **전체 텍스트 추출** (PyPDF2 등):
   ```python
   from PyPDF2 import PdfReader
   r = PdfReader(path)
   for i,p in enumerate(r.pages): print('== PAGE', i+1, '=='); print(p.extract_text())
   ```
3. **인포그래픽 페이지는 이미지로 렌더링**해 시각 구조(카드 묶음·표·플로우)를 확인. 텍스트 추출은 레이아웃·표 관계를 잃기 때문이다.
   ```python
   import fitz   # PyMuPDF
   doc = fitz.open(path)
   doc[n].get_pixmap(matrix=fitz.Matrix(1.4,1.4)).save(out)
   ```
   (pdftoppm/Read 이미지가 막히면 PyMuPDF 또는 pdf2image로 PNG 저장 후 이미지로 읽는다.)

## 무엇을 뽑는가
- **섹션 트리**: 원문 위계(Ⅰ, Ⅱ-1 …) 그대로. 각 노드에 핵심메시지·하위항목·표/예시·출처페이지.
- **인터랙티브 후보**: 점검표/설문 → 점수형 퀴즈, 다짐/약속 → 저장형 체크리스트, 프롬프트 예시 → 복사형 스니펫, 위험·한계 목록 → 아코디언/카드, 단계(STEP·STOP) → 플로우. 각 후보의 필요 데이터를 함께.
- **정확성 보존 데이터**: 설문 문항 전체, 점수 구간 경계, 약속 항목, 전화번호 등은 **원문 글자 그대로** 별도 정리(이후 단계에서 변형 방지).

## 원칙 (왜)
- 요약은 하되 **사실·수치·예시 문구는 보존**한다. 교육자료의 정확성이 콘텐츠 충실성의 핵심이기 때문.
- 섹션을 묶거나 나눈 **근거를 한 줄씩** 남긴다 — 다음 단계가 IA를 설계하는 입력이 된다.
