# 충실성 리뷰 (생성-검증)

빌더와 **분리된 패스**로 완성물을 원문/명세와 대조한다. 핵심은 "존재 확인"이 아니라 **경계면 교차 비교**다.

## 4대 검증축
1. **콘텐츠 충실성** — `index.html`의 데이터 배열 ↔ `01_content-map.md`의 보존 데이터를 항목 단위로 대조. 설문 문항 수·문구, 점수밴드 경계, 약속 항목 수·문구, 전화번호, 단계(STOP) 문구가 누락·왜곡 없이 일치하는가.
2. **인터랙션 로직** — 코드 레벨 확인:
   - 퀴즈 합산식이 `Σ(1~k점)`이고 만점=`문항수×k`인가. 결과밴드 경계(이상/이하)가 원문과 일치하고 겹침/구멍이 없는가.
   - localStorage 키가 읽기/쓰기에서 동일한가. 첫 진입 null 방어가 있는가.
   - 라디오 그룹 name이 문항별로 고유한가. 미응답 시 결과 확정을 막는가.
   - 복사 버튼이 올바른 인덱스의 텍스트를 복사하는가.
3. **디자인 일치** — 3px 테두리·하드 섀도우(blur 0)·섹션당 단색·굵은 제목·Pretendard·점 패턴 배경.
4. **접근성·반응형** — 대비, 터치 타깃(≥44px), aria/포커스, 모바일·전자칠판 레이아웃, 표 가로 스크롤.

## 정적 점검 보조
가능한 것은 grep/스크립트로 객관화한다. 예: 설문 문항 배열 길이 == 원문 문항수, `localStorage.` 키 문자열 일치 여부, `box-shadow` 중 blur 값(3번째 0px) 검사.

## 심각도
- **P0**: 콘텐츠 누락/왜곡, 점수 합산·밴드 오류, 저장 동작 불가
- **P1**: 접근성·반응형 결함
- **P2**: 디자인 디테일 불일치
- **P3**: 개선 제안

## 출력
`_workspace/04_review.md` — 판정(PASS/REVISE/REWRITE) + 심각도별 목록(항목·위치·근거·수정안) + 통과 확인 목록. REVISE/REWRITE면 P0/P1을 web-builder에 넘겨 재수정. PASS까지 최대 2회 루프 권장.
